# 커밋 메시지 생성

다음 단계를 따라 팀 규칙에 맞는 커밋 메시지를 생성해주세요:

## 1단계: 변경사항 분석
- `git diff --staged` 명령어로 스테이징된 변경사항 확인
- 변경된 파일과 코드 내용 파악

## 2단계: 커밋 타입 결정
다음 중 적절한 타입 선택:
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 코드 리팩토링 (기능 변경 없음)
- `style`: 코드 포맷팅, 세미콜론 누락 등
- `docs`: 문서 수정
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드 업무, 패키지 매니저 설정 등

## 3단계: 커밋 메시지 작성 규칙
- **제목**: `[타입] 한글로 작성` (50자 이내)
- **본문** (선택사항):
  - 무엇을 변경했는지
  - 왜 변경했는지
  - 관련 이슈 번호 (있는 경우)

## 4단계: 커밋 메시지 제안
위 규칙에 따라 3가지 옵션을 제안해주세요:
1. 간결한 버전 (제목만)
2. 표준 버전 (제목 + 간단한 본문)
3. 상세 버전 (제목 + 상세 본문)

## 예시
```
[feat] 사용자 로그인 API 추가

- JWT 기반 인증 구현
- 로그인 실패 시 에러 처리 추가
- 관련 이슈: #123
```

**주의사항**:
- 변경사항이 없으면 커밋하지 마세요
- 여러 기능이 섞여있다면 커밋 분리를 권장합니다
- .cursorrules 파일의 커밋 규칙을 준수하세요
